# Design Notes

## 見直し方針

Apple Human Interface Guidelinesを参照し、スマホPWAでもiOSアプリに近い基本体験になるように再設計した。

- 階層: 画面タイトル、主要タスク、補助情報の順に整理
- 一貫性: 下部タブは4項目に絞り、画面ごとに同じ位置に維持
- 読みやすさ: 日本語システムフォント、17px基準、太字を避けたウェイト、十分な行間
- タップしやすさ: ボタン、チップ、入力、ナビを44px以上の操作領域に統一
- 色: 低彩度グレー背景、深めのティール主操作、緑/橙/赤は意味を限定し、主要コントラストは4.5:1以上
- 入力削減: 今日画面は4ステップに分割し、状態チップと完了ボタンだけに絞る
- テーマ: ライト、ダーク、システム追従を設定から即時切り替え
- 適応性: 横スクロールを禁止し、モバイル/デスクトップの両方で再配置
- PWA: standalone表示、ショートカット、maskable icon、オフライン起動を維持

## ツール構成

- 今日: いま書くための入口。1.型を選ぶ、2.紙に写す、3.書く、4.書けたを1つずつ表示
- 型: 紙に写すテンプレートライブラリ
- 習慣: if-thenトリガー、14日プログラム、週次レビュー、安全チェック、履歴
- 設定: 紙ノート名、標準時間、画面テーマ、本文非保存、データ入出力

## 検証項目

- JS構文チェック
- ユニットテスト
- PWA manifest / icons / shortcuts
- Service Workerオフライン表示
- モバイル全タブの横スクロールなし
- 主要操作対象の44px以上タップ領域
