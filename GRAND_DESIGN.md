# Grand Design

## 設計原則

AppleのUI Design Dos and Don'tsでは、主要コンテンツが横スクロールやズームなしで見えること、タッチ操作に適したUI要素を使うこと、44pt四方以上のヒットターゲット、十分なコントラスト、テキストの重なりを避ける余白、関連する操作を対象コンテンツの近くに置くことが示されている。W3C WCAG 2.2でも、モバイル利用を含めてターゲットサイズや一貫した操作を重視している。

このアプリでは次の設計原則に落とし込む。

- まず書く: 起動直後の主目的は「今日の1回を始める」こと
- 迷わせない: トップレベルは4タブに絞る
- 紙を主役にする: アプリは本文入力欄を持たず、問いと時間だけを出す
- 疲れた日を守る: 1分レスキューを今日画面の上部に固定する
- 脳内翻訳をなくす: 本番中の入力欄を増やさず、押すだけで進める
- タイマーを迷わせない: 書く時間は大きな数字と短い操作だけで扱う
- 重い見た目を避ける: 太字、濃色カード、装飾、巨大な図形を減らす
- 太字で読ませない: 見出しも400台のウェイトに抑え、余白と面で階層を作る
- 行動と記録を近づける: 完了ボタンをタイマー直後に置く
- 週次は習慣に統合する: 毎日見る必要がない機能はトップタブから外す

## 情報設計

### タブ構成

1. 今日
   - 今日の状態
   - 1分レスキュー
   - 状態チップ
   - 紙に写す問い
   - タイマー
   - 書けたボタン

2. 型
   - テンプレート一覧
   - 目的別フィルター
   - 今日の型に設定

3. 習慣
   - 直近リズム
   - if-thenトリガー
   - 14日プログラム
   - 週次レビュー
   - 最近の記録

4. 設定
   - ノート名
   - 標準時間
   - ライト / ダーク / システム追従
   - 本文非保存
   - エクスポート / インポート / 削除

## 画面設計

### 今日画面

今日タブはまず「準備」だけを表示する。今日のヒント1行と30秒整理で頭を整えてから「1. 型を選ぶ → 2. 紙に写す → 3. 書く → 4. 書けた」に進む。各ステップは縦に並べず、必要な時にだけ切り替える。本番中の入力欄は増やさず、状態チップと完了ボタンだけで実行記録が残る構造にする。

### 型画面

テンプレートはカードを大きくしすぎず、タイトル、用途、短い説明、設定ボタンだけにする。説明文は本文ウェイトにして、太字を抑える。

### 習慣画面

数値は小さなメトリクス行に集約する。14日ドットは2行表示にして横スクロールを防ぐ。週次レビューは同じ画面内に置き、トップタブから外す。

## ビジュアル方針

- 背景: 低彩度の静かなグレー
- 表面: 白またはわずかに透過した白
- 主色: 深めの低彩度ティール
- 補助色: 緑は安全、橙はレスキュー、赤は削除だけ
- 文字: 17px基準、本文と見出しは300前後
- 角丸: 大きすぎない14px前後
- 影: ほぼ使わず、境界線と余白で階層化
- コントラスト: 本文、補助テキスト、主ボタンは 4.5:1 以上

## 検証基準

- 390px幅で横スクロールなし
- 主要ボタン、チップ、入力、タブが44px以上
- 今日タブ初期表示で見える主作業は型選択だけにする
- プロンプトが折り返しても重ならない
- 今日画面に本文や気分の入力欄がない
- PWA manifest、PNGアイコン、オフライン起動が動作する
